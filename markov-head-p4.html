马尔可夫头 — 视觉化卡片（ljg-card 风格）

**概念压缩**：马尔可夫头是 DSpark 推测解码系统中的轻量级串行修正模块，仅依赖前一个 token 的嵌入，通过 rank-256 低秩分解生成修正 logits，以几乎为零的计算成本将草稿接受长度提升 ~30%。

│                                                                 │
│  修正公式：                                                      │
│  ┌──────────────────────────────────────────────────────┐       │
│  │  Δlog p(q_i | q_{i-1}) = W_B · σ(W_A · embed(q_{i-1}))  │  │
│  └──────────────────────────────────────────────────────┘       │

│  │  W_B             ∈ ℝ^{128000 × 256}  (上投影)         │       │
│  │  Δlog p          ∈ ℝ^{128000}        (修正增量)       │       │
│  └──────────────────────────────────────────────────────┘       │
│                                                                 │

│  ┌──────────────────────────────────────────────────────┐       │
│  │  Params = 256×4096 + 128000×256 = 33.8M              │       │

│  └──────────────────────────────────────────────────────┘       │

│  为什么 rank 256 就够了？                                        │

│  │  ||f - f_r||_F ≤ σ_{r+1}(f)                          │       │
│  │  当 r=256 时，捕获 95%+ 修正能量                       │       │
│  │  奇异值衰减极快：前 256 个 SV 已足够                    │       │
│  └──────────────────────────────────────────────────────┘       │

卡片 6：代码速查

核心实现（5 行代码）


# 马尔可夫头的核心计算（5 行）
class MarkovHead(nn.Module):
    def __init__(self, d_embed=4096, vocab_size=128000, rank=256):
        self.down = nn.Linear(d_embed, rank, bias=False)  # 4096 → 256
        self.up   = nn.Linear(rank, vocab_size, bias=False)  # 256 → 128K
        self.act  = nn.SiLU()
    
    def forward(self, prev_embed):
        return self.up(self.act(self.down(prev_embed)))
        # prev_embed: [B, 4096] → delta_logits: [B, 128000]
        # FLOPs: ~67.6M | Params: ~33.8M

│  ┌──────────────────────────────────────────────────────┐       │

│  适用条件：                                                      │
│  • 基础预测已经"大致正确"                                        │
