+title: 概念解剖：多Token预测 (Multi-Token Prediction)

+filetags: :concept:ai:training:

+date: [2026-06-28]

  • 定锚

多Token预测（MTP）：大语言模型训练和推理的范式革新。在训练阶段，模型不仅学习预测下一个 token，还同时学习预测未来第 2、3、...、N 个 token。推理时，这些预测头直接作为推测解码的草稿生成器，实现训练-推理协同优化。本质是"从逐步猜测到批量预知"。

常见误解： - 误解一："MTP 就是推测解码"——错。MTP 是训练方法，推测解码是推理框架。MTP 是推测解码的一种草稿生成方式。 - 误解二："MTP 增加了太多训练成本"——实际只增加 5-10%。MTP 模块轻量（约占总参数 1-2%）。 - 误解三："MTP 会损害模型质量"——不会。适当的损失加权下，MTP 甚至可能提高质量（更丰富的表示学习）。 - 误解四："MTP 只适用于 DeepSeek 架构"——错。DSpark 在 Qwen3 上也实现了 30% 加速，证明了通用性。 - 误解五："MTP 深度越大越好"——错。N 过大会导致优化困难和接受率下降。最优 N 通常 3-5。

核心词素： - Multi（多）：多个未来 token - Token（标记）：语言模型的基本处理单元 - Prediction（预测）：从条件分布中推断未来 - 隐含词素：Training-Inference Co-design（训练-推理协同设计） - 隐含词素：Speculative Decoding（推测解码）——MTP 的主要应用场景 - 隐含词素：Conditional Dependency（条件依赖）——MTP 有效的前提

  • 八刀

** 历史

2003 年，Bengio 发表《A Neural Probabilistic Language Model》，确立了 next-token prediction 作为语言模型的标准训练目标。论文简要讨论了"预测多个未来 token"的可能性，但认为计算成本过高而放弃。这个被放弃的方向沉睡了二十年。

2015 年，Bengio 自己提出 Scheduled Sampling，解决训练-推理不匹配（exposure bias）问题。这可以看作多步预测的早期形式——模型在训练时就开始学习处理自己预测的未来 token。

2018 年，Stern 等人发表 Blockwise Parallel Decoding，首次将多步预测用于推理加速。但辅助模型是独立训练的浅层模型，与主模型没有深度集成。

2024 年是 MTP 的突破年。DeepSeek-V3 首次在工业级模型中集成 MTP 训练目标——在预训练阶段就加入多 token 预测损失。同年，Medusa 在目标模型上添加多个解码头（推理时微调），EAGLE 在特征空间做多步预测。

2025 年，DSpark 将 MTP 从训练优化提升为推理加速的核心引擎。在 DeepSeek-V4-Flash/Pro 上实现 60-85% 加速，在 Qwen3 上实现 30% 加速。MTP 的跨模型通用性得到验证。

更深层的历史脉络：MTP 是"训练-推理协同设计"大趋势的开端。它打破了"训练时不想推理"的传统范式，开创了"训练时就为推理做准备"的新范式。

** 辩证

正面：MTP——训练时学会多步预测，推理时直接复用。 反面：MTP 增加了训练复杂性——额外的损失函数、超参数、模块设计。 碰撞后的更高理解：训练-推理协同设计。不是"训练 vs 推理"的二选一，而是"训练时考虑推理，推理时复用训练"。MTP 的额外训练成本（5-10%）换来推理效率的大幅提升（60-85%），这是一笔划算的交易。

更深层的辩证：MTP 揭示了 NTP 范式的一个根本性低效——每次前向传播只产出 1 个 token 的监督信号。这个低效在模型小的时候不重要（计算便宜），但在模型大的时候变得不可接受（计算昂贵）。MTP 通过让每次前向传播产出 N 个 token 的监督信号，将训练效率提升了 N 倍。但这也引出了一个问题：如果 MTP 这么好，为什么 NTP 统治了三十年？答案是：NTP 更简单、更稳定、更容易优化。MTP 的额外复杂性在小模型时代不值得，但在大模型时代变得必要。

** 现象

想象一个棋手。新手（NTP）每走一步都要停下来想很久——只看眼前这一步的后果。高手（MTP）一眼就能看到未来好几步——"如果我走这里，对手大概率走那里，然后我再走这里……"。高手不是每步想得更快，而是他同时看到了多个可能的未来。

MTP 就是把语言模型从新手变成高手。不是让它每步算得更快，而是让它同时看到多个未来。这种"批量预知"的能力不仅让推理更快（因为可以跳过已验证的步骤），还可能让模型更"聪明"（因为它在训练时就学会了考虑更长远的依赖关系）。

** 语言

  • Multi：源自拉丁语 =multus=（多），暗含"不止一个"的意味。
  • Token：源自希腊语 =tokenos=（标记），在 AI 语境中指语言模型的基本处理单元。
  • 语义网：
  • Next-Token Prediction（NTP）：MTP 的前身和对照
  • Speculative Decoding（推测解码）：MTP 的主要应用场景
  • Training-Inference Co-design（训练-推理协同设计）：MTP 开创的范式
  • Exposure Bias（暴露偏差）：MTP 可以缓解的训练问题
  • Conditional Dependency（条件依赖）：MTP 有效的理论前提
  • 暗含隐喻："看未来"。这个隐喻挡住了一个问题：MTP 不是真的"看到"未来，而是学会了未来 token 之间的条件依赖关系。

** 形式

核心公式：

训练损失：=L_total = L_NTP + alpha * sum_{k=2}^{N} L_MTP_k=

MTP 模块：=p_k(token_{t+k} | h_t, token_{t+1}, ..., token_{t+k-1})= 其中 =h_t= 是主模型的隐藏状态，=p_k= 是第 k 个 MTP 预测头

接受率理论：=accept_rate_MTP >= 1 - KL(p_MTP || p_NTP)=

最优 MTP 深度：=N* = argmax_N { speedup(N) * accept_rate(N)^N }=

失效边界： - MTP 深度 N 过大时，远期 token 的预测不准确，接受率下降 - alpha 过大时，MTP 损失主导训练，可能损害 NTP 质量 - 当 token 之间条件独立时（如随机序列），MTP 没有价值

** 存在

MTP 改变了模型训练的方式。从"只优化训练损失"到"同时考虑推理效率"。这种思维方式的转变可能比 MTP 本身更有价值——它开创了"训练-推理协同设计"的新范式。

MTP 也改变了推理效率的经济学。传统推理优化（量化、蒸馏、剪枝）都是有损的——用质量换速度。MTP 通过训练时的少量投入（5-10% 训练成本增加），换取推理时的大幅加速（60-85%）。这是一种"投资"而非"牺牲"。

** 美感

美在统一。MTP 将训练和推理统一为一个整体——训练时的多步预测能力直接用于推理时的推测加速。没有"训练完再优化"的割裂，没有"训练目标与推理目标不一致"的矛盾。训练即推理，推理即训练。

美在效率。NTP 每次前向传播只产出 1 个监督信号，MTP 产出 N 个。这是 N 倍的效率提升——不是通过更多的计算，而是通过更充分地利用已有的计算。就像一棵树不仅结一种果子，还同时结多种果子——阳光和土壤的利用效率更高了。

  • 缝合

MTP 与推测解码的关系：MTP 是推测解码的最优草稿来源。推测解码是框架，MTP 是框架中的一种草稿生成方法。MTP 草稿质量最高（共享表示），但需要训练时集成。

MTP 与 Scaling Law 的关系：Scaling Law 推动模型越来越大，推理越来越慢，推测解码需求越来越迫切，MTP 作为推测解码的最优草稿来源，价值越来越大。

MTP 与 MoE 的关系：MTP 模块可以与 MoE 结合——MTP 模块只激活部分专家，减少额外计算成本。DeepSeek-V3 的 MoE + MTP 组合是这种结合的范例。

MTP 与自动驾驶的关系：驾驶场景的时序连续性使得 MTP 在自动驾驶中特别有效——接受率可达 85-90%，高于文本推测解码的 70-80%。

  • 延伸：MTP 的数学本质

从信息论角度看，MTP 的本质是利用了 token 序列中的冗余信息。自然语言具有高度的统计冗余——给定前面的上下文，未来多个 token 的不确定性是递减的。标准 NTP 只消除了第 1 个未来 token 的不确定性，MTP 同时消除了多个未来 token 的不确定性。

从表示学习角度看，MTP 迫使模型学习更丰富的表示。标准 NTP 只需要表示"当前上下文"，MTP 需要表示"当前上下文的未来演化"。后者包含了更多的信息——模型不仅要理解当前在说什么，还要理解接下来可能说什么。这种更丰富的表示可能带来更好的泛化能力。

从优化角度看，MTP 提供了更多的梯度信号。标准 NTP 每步只产生 1 个 token 的梯度，MTP 每步产生 N 个 token 的梯度。更多的梯度信号意味着更高效的训练——每个 mini-batch 的信息利用率更高。这可能是 MTP 训练成本只增加 5-10% 但效果显著的原因。

  • 延伸：MTP 与其他训练范式的关系

MTP 与课程学习（Curriculum Learning）的关系：可以先训练 N=1（标准 NTP），再逐步增加 N（MTP 深度）。这种"课程式 MTP"可能比直接训练大 N 更稳定。

MTP 与对比学习（Contrastive Learning）的关系：MTP 预测的是"正确的未来 token"，可以引入"错误的未来 token"作为负样本，形成对比学习框架。这可能进一步提高 MTP 模块的判别能力。

MTP 与强化学习（RL）的关系：MTP 预测多个未来 token，可以看作是一种"想象"——模型在内部模拟未来的多种可能。这与强化学习中的"模型预测控制"（MPC）有相似之处——都是通过在内部模拟未来来指导当前决策。

  • 延伸：MTP 的失败模式与对策

MTP 并非总是有效。理解其失败模式对于正确应用 MTP 至关重要：

失败模式一：分布偏移。当推理时的输入分布与训练时差异较大时（如遇到训练数据中罕见的主题），MTP 模块的预测质量会大幅下降。对策：引入分布检测机制，当检测到分布偏移时自动降低 MTP 深度或关闭 MTP。

失败模式二：误差累积。MTP 深度 N 较大时，远期 token 的预测误差会累积。特别是当第 k 个 token 的预测错误导致第 k+1 个 token 的预测条件发生变化时，误差会雪崩式放大。对策：在 MTP 模块中引入"纠错"机制——第 k 个预测头的输入包含前 k-1 个预测头的实际输出（而非预测输出），形成闭环校正。

失败模式三：任务冲突。某些任务（如创意写作）需要高多样性的输出，MTP 的多步预测可能抑制多样性——因为 MTP 倾向于预测"最可能"的未来，而非"最有趣"的未来。对策：在创意任务中降低 MTP 的损失权重 alpha，让 NTP 目标主导训练。
