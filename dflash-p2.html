DFlash 核心论文深度解读

本文系统梳理 DFlash 及其相关工作的核心论文，涵盖并行推测解码、扩散语言模型、DSpark 等关键研究方向。

论文一：DFlash — Block Diffusion for Flash Speculative Decoding

基本信息

  • **标题**：Block Diffusion for Flash Speculative Decoding
  • **作者**：DeepSeek-AI 团队
  • **发表时间**：2026 年 2 月
  • **发表会议**：ICML 2026
  • **论文地址**：arxiv.org/abs/2602.06036
  • **开源代码**：github.com/z-lab/dflash

    • 核心问题

    自回归语言模型的推理速度受限于其串行解码机制：每个 token 必须等待前一个 token 生成完毕。标准推测解码（Speculative Decoding）通过引入草稿模型加速生成，但草稿模型本身仍是自回归的，成为新的速度瓶颈。

    核心问题：能否让草稿模型也实现完全并行，一次前向传播生成所有候选 token？

    核心方法

    DFlash 提出使用块扩散（Block Diffusion）模型作为草稿模型，替代传统的自回归草稿模型。

    1. 块扩散机制

    块扩散的核心思想是将 token 生成建模为去噪过程：

    1. 从纯噪声开始（所有位置都是随机 token）
    2. 逐步去噪，将噪声位置替换为高置信度 token
    3. 经过固定步数（通常 1 步）后，输出完整的 token 块

数学形式化：

设当前上下文为 $x_{1:t}$，需要生成后续 $K$ 个 token。块扩散模型学习条件分布：

$$P_\theta(y_{t+1:t+K} | x_{1:t})$$

其中 $y_{t+1:t+K}$ 通过扩散过程生成：

$$y^{(0)} \sim \text{Uniform}(\mathcal{V})$$

$$y^{(s)} = f_\theta(y^{(s-1)}, x_{1:t}, s)$$

$y^{(0)}$ 为初始噪声（从词表 $\mathcal{V}$ 中均匀采样），$y^{(s)}$ 为第 $s$ 步去噪后的结果，$f_\theta$ 为参数化的去噪网络。

2. 模型架构

DFlash Draft 模型采用双向注意力 Transformer：

  • **6 层 Transformer**：参数量远小于目标模型
  • **双向注意力**：`is_causal=False`，每个位置可以看到所有其他位置
  • **输入构成**：
  • `target_hidden`：目标模型最后一层的隐藏状态
  • `noise_embedding`：当前 draft token 的嵌入（初始为噪声）
  • `position_ids`：位置编码
    • 
# DFlash 核心前向传播
textnoise_embedding = target.model.embed_tokens(block_output_ids)
draft_logits = target.lm_head(model(
    target_hidden=target_hidden,

    past_key_values=past_key_values_draft,
    use_cache=True,
    is_causal=False,  # 双向注意力的关键
))[:, 1 - block_size:, :])

    3. 训练目标

    DFlash 的训练目标是最大化给定上下文下 token 块的对数似然：

    $$\mathcal{L} = -\mathbb{E}_{x, y} \left[ \sum_{i=1}^{K} \log P_\theta(y_{t+i} | x_{1:t}, y_{t+1:t+i-1}^{\text{target}}) \right]$$

    其中 $y^{\text{target}}$ 为目标模型生成的真实 token。

    4. 与目标模型的协同

    DFlash 的关键设计决策：

    • **输入来自目标模型**：DFlash 接收目标模型的隐藏状态和 KV Cache 作为上下文
    • **输出由目标模型验证**：DFlash 生成的 draft token 由目标模型通过拒绝采样验证
    • **质量兜底**：即使 DFlash 生成质量不高，目标模型的验证保证了最终输出的无损性

      核心贡献

      2. **完全并行的草稿生成**：一次前向传播生成 16 个候选 token
      3. **无损加速保证**：通过拒绝采样确保输出分布与目标模型一致

    4. 广泛的模型兼容性：支持 vLLM、SGLang、Transformers、MLX 等主流框架

    局限性

    1. **多模态碰撞**：各位置独立采样导致不通顺的 token 组合
    2. **后缀衰减**：位置越靠后，接受率越低
    3. **训练数据依赖**：需要目标模型的隐藏状态作为训练信号

4. 复杂推理场景：对于接受率本身很低的复杂请求，并行草稿阶段的计算是浪费的

论文二：DSpark — Confidence-Scheduled Speculative Decoding with Semi-Autoregressive Generation

基本信息

  • **标题**：DSpark: Confidence-Scheduled Speculative Decoding with Semi-Autoregressive Generation
  • **作者**：DeepSeek-AI, 北京大学
  • **首位作者**：袁境阳（北大博士），导师：张铭教授
  • **发表时间**：2026 年 6 月 27 日
  • **开源代码**：github.com/deepseek-ai/DeepSpec（MIT 许可）
  • **部署状态**：已部署于 DeepSeek-V4-Flash 和 DeepSeek-V4-Pro 生产环境

    • 核心问题

    DFlash 虽然实现了极快的草稿生成速度，但暴露了两个核心问题：

    1. **多模态碰撞**：并行采样导致不通顺的 token 组合
    2. **后缀衰减**：位置越靠后，接受率急剧下滑

核心问题：能否在保持并行速度的同时，解决多模态碰撞和后缀衰减问题？

核心方法

DSpark 提出两大互补机制：半自回归生成和置信度调度验证。

1. 半自回归生成（Semi-Autoregressive Generation）

核心思想：不抛弃并行，但加上一个轻量的"依赖注入"模块。

架构设计：

  • **并行主干（DFlash）**：一次前向传播生成所有位置的基础 logits
  • **轻量级顺序头（Markov Head / RNN Head）**：在采样时注入前缀依赖信息

Markov Head 设计：


class MarkovHead(nn.Module):
    def __init__(self, hidden_dim=256, rank=256):
        super().__init__()
        self.proj_down = nn.Linear(hidden_dim, rank, bias=False)
        self.proj_up = nn.Linear(rank, hidden_dim, bias=False)
    
    def forward(self, hidden_states, prev_token_ids):
        # 获取前一个 token 的嵌入
        prev_embed = self.embedding(prev_token_ids)
        # 低秩投影
        prev_feat = self.proj_down(prev_embed)
        # 注入到当前隐藏状态
        injection = self.proj_up(prev_feat)
        return hidden_states + injection

关键设计：

  • 256 维低秩分解：参数量极小，几乎不增加计算开销
  • 仅注入"前一个 token 是什么"的信息：让后续 token 的预测不再是盲猜
  • 端到端延迟增加仅 0.6%-1.3%（draft 长度从 4 扩到 16）

实验效果：

位置 DFlash 接受率 DSpark 接受率 改善
1 0.72 0.82 +13.9%
7 0.63 0.79 +25.4%

关键发现：仅 2 层 DSpark 的性能就超过 5 层 DFlash，说明一点点自回归信息注入的价值巨大。

2. 置信度调度验证（Confidence-Scheduled Verification）

核心思想：根据每个 draft token 的置信度和系统负载，动态决定验证长度。

组件一：置信度估计头

额外训练一个置信度头，预测每个 draft token 的"前缀存活概率"——即如果前面的 token 都通过了验证，这个位置的 token 还能活下来的概率。

数学形式化：

$$c_i = P(\text{accept}_i | \text{accept}_{

组件二：顺序温度缩放（Sequential Temperature Scaling, STS）

神经网络天生会过度自信。STS 通过校准方法将预测概率和真实接受率对齐：

$$\hat{c}_i = \sigma\left(\frac{\log c_i}{T_i}\right)$$

其中 $T_i$ 为位置相关的温度参数，通过验证集学习得到。

校准效果：

方法 平均校准误差（ECE）
原始置信度 3%-8%
标准温度缩放 2%-5%
**STS** **约 1%**

组件三：硬件感知前缀调度器

综合考虑系统负载和每个 token 的生存概率，动态决定每个请求该验证几个 token：

$$L^* = \arg\max_L \mathbb{E}\left[\sum_{i=1}^{L} \hat{c}_i \right] \quad \text{s.t.} \quad \text{GPU\_load} < \text{threshold}$$

调度策略：

  • 轻负载时：大胆多验证（4-6 个 token）
  • 高并发时：果断剪掉低置信后缀（2-3 个 token）
  • 动态调整：根据实时 GPU 负载自适应

关键约束：不能提前偷看未来 token，否则会破坏无损推测解码的理论保证。DSpark 通过异步调度（使用两步前的历史预测来确定裁剪长度）巧妙绕开了这个问题。

      4. 论文三：Speculative Decoding（标准推测解码）

      基本信息

    4. **标题**：Fast Inference from Transformers via Speculative Decoding
    5. **作者**：Google Research
    6. **发表时间**：2023 年
    7. **发表会议**：ICML 2023

      8. 核心方法

    标准推测解码的核心思想：

    1. **Draft 阶段**：使用轻量级草稿模型自回归生成 $K$ 个候选 token
    2. **Verify 阶段**：目标模型并行验证这 $K$ 个 token
    3. **Accept/Reject**：通过拒绝采样决定接受哪些 token

      4. 拒绝采样机制：

      对于每个位置 $i$，计算接受概率：

      $$A_i = \min\left(1, \frac{P_{\text{target}}(y_i | y_{

      以概率 $A_i$ 接受 $y_i$，否则拒绝并从 $P_{\text{target}}$ 重新采样。

      无损性保证：通过拒绝采样，推测解码的输出分布在数学上严格等价于直接使用目标模型解码。

      核心贡献

      1. **提出推测解码范式**：开创了"猜测-验证"的推理加速新赛道
      2. **数学证明无损性**：拒绝采样保证输出分布不变
      3. **实现 2-3x 加速**：在多个任务上验证效果

        4. 局限性

      4. **草稿模型瓶颈**：草稿模型本身是自回归的，速度有上限
      5. **加速比有限**：通常只能实现 2-3x 加速
      6. **需要额外模型**：需要训练或选择独立的草稿模型

        7. 论文四：Medusa — Simple LLM Inference Acceleration with Multiple Decoding Heads

        基本信息

      7. **标题**：Medusa: Simple LLM Inference Acceleration with Multiple Decoding Heads
      8. **作者**：UC Berkeley
      9. **发表时间**：2024 年
      10. **发表会议**：ICML 2024

        11. 核心方法

      Medusa 的核心创新：

      1. **多头预测**：在原始 LLM 基础上增加多个预测头，每个头独立预测未来不同位置的 token
      2. **树形注意力**：使用树形注意力机制实现并行验证
      3. **典型接受**：引入基于阈值的典型接受策略

        4. 数学形式化：

        设模型最后一层隐藏状态为 $h_t^{(L)}$，Medusa 添加 $K$ 个预测头：

        $$\hat{y}_{t+k} = \text{Head}_k(h_t^{(L-1)}), \quad k = 1, 2, \ldots, K$$

        每个头独立预测未来第 $k$ 个位置的 token。

        核心贡献

        1. **消除外部草稿模型**：减少内存占用，简化部署
        2. **多头并行预测**：无需额外前向传播
        3. **树形注意力验证**：提升验证效率

          4. 局限性

        4. **预测头之间缺乏交互**：接受率随距离递减
        5. **单次草稿量有限**：通常只能预测 3-5 个 token
        6. **训练开销**：需要训练多个预测头

          7. 论文五：EAGLE — Extrapolation Algorithm for Greater-speed Lossless Efficiency

          基本信息

        7. **标题**：EAGLE-3: Accelerating Autoregressive Generation via Speculative Decoding with Dynamic Draft Tree
        8. **作者**：多个版本迭代
        9. **发表时间**：2024-2025 年

          10. 核心方法

        EAGLE 系列的核心创新：

        EAGLE-1：将推测解码前移至特征层

        • 对隐藏状态进行自回归预测，而非直接预测 token
        • 数学形式化：$\hat{f}_{t+k} = g(f_{t}, f_{t-1}, \ldots, f_{t-k+1})$
        • 其中 $f_t$ 为第 $t$ 步的隐藏状态

        EAGLE-2：自适应草稿树

        • 根据上下文动态调整草稿结构
        • 使用树形注意力机制验证

        EAGLE-3：优化训练目标

        • 优化训练目标，实现更高接受率
        • 动态草稿树生成

          • 核心贡献

        • **特征层推测**：将推测解码前移至隐藏状态层面
        • **自适应草稿树**：动态调整草稿结构
        • **高接受率**：通过特征级预测提升接受率

          • 局限性

        • **草稿生成仍为串行**：虽然是在特征层面，但仍然是自回归的
        • **速度提升有上限**：受限于串行草稿生成
