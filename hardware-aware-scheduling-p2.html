硬件感知调度与在线校准 — 核心论文深度解读

**论文脉络**：从推测解码的基础框架，到自适应草稿长度的探索，再到硬件感知调度和在线校准的最终提出——一条从"加速推理"到"智能调度"的学术演化路径。

一、奠基论文：推测解码与自适应长度

1.1 Leviathan et al. (2023) — 推测解码框架

论文：Fast Inference from Transformers via Speculative Decoding

核心贡献：建立了推测解码的理论框架，定义了草稿-验证范式。理论加速比公式：

$$

S = \frac{\alpha K + 1}{T_{\text{draft}} \cdot K + T_{\text{verify}}}

$$

局限性：假设固定草稿长度 $K$，未考虑自适应调整。

1.2 Chen et al. (2023) — 推测采样

论文：Accelerating Large Language Model Decoding with Speculative Sampling

核心贡献：独立提出推测解码，给出了完整的拒绝采样算法和正确性证明。关键定理：

$$

\mathbb{E}[\text{accept}] = 1 - \text{TVD}(p_{\text{draft}}, p_{\text{target}})

$$

与调度的关系：接受率直接取决于分布差异，而分布差异随位置变化——这为自适应长度提供了理论基础。

1.3 SpecInfer (2023) — 树状推测

论文：SpecInfer: Accelerating Generative LLM Serving with Tree-based Speculative Inference

核心贡献：提出树状草稿结构，通过集成多个小模型提升草稿多样性。

与调度的关系：树状结构引入了"验证多少"的新维度——不仅可以选择链的长度，还可以选择树的形状。

二、自适应长度论文

2.1 EAGLE (Li et al., 2024)

论文：EAGLE: Speculative Sampling Requires Rethinking Feature Level

核心贡献：

  1. 特征级推测：在目标模型中间层提取特征
  2. 自适应草稿长度：基于置信度阈值动态调整

    3. 自适应策略：

    $$

    K_{\text{EAGLE}} = \min\{k : P(q_k) < \theta_{\text{conf}}\}

    $$

    当草稿模型的 top-1 概率低于阈值 $\theta_{\text{conf}}$ 时停止生成。

    局限性：只看草稿模型的置信度，不考虑硬件状态。

    2.2 Medusa (Cai et al., 2024)

    论文：Medusa: Simple LLM Inference Acceleration with Multiple Decoding Heads

    核心贡献：多个 MLP 头同时预测多个未来位置，通过树状注意力验证。

    与调度的关系：Medusa 的头数决定了最大预测步数，但实际验证长度仍然是固定的。

    2.3 Sarathi-Serve (2024)

    论文：Sarathi-Serve: Efficient LLM Serving with Chunked Prefills and Speculative Decoding

    核心贡献：

    1. Chunked Prefill：将长 prompt 分成多个 chunk 处理
    2. 与推测解码结合：在 decode 阶段插入 prefill chunk

      3. 调度策略：固定 chunk 大小，根据队列长度动态调整 prefill/decode 比例。

      局限性：

      • chunk 大小固定，不感知硬件状态
      • 需要 CPU 参与调度决策
      • 不考虑草稿模型的置信度

        • 三、校准理论论文

        3.1 Temperature Scaling (Guo et al., 2017)

    论文：On Calibration of Modern Neural Networks

    核心贡献：

    1. 现代神经网络普遍存在校准问题
    2. 温度缩放是简单有效的校准方法
    3. ECE 作为校准质量的标准度量

      4. 温度缩放公式：

      $$

      p_{\text{calibrated}} = \text{softmax}(z / T), \quad T^* = \arg\min_T \text{NLL}(z/T, y)

      $$

      3.2 Sequential Calibration (Zhang et al., 2024)

      论文：Sequential Temperature Scaling for Speculative Decoding

      核心贡献：

      1. 发现推测解码中不同位置的校准误差模式不同
      2. 提出顺序温度缩放：每个位置一个温度参数
      3. 证明顺序校准可以将 ECE 从 5% 降到 ~1%

        4. 关键发现：

        位置 最优温度 标准温度 校准误差
        1 1.02 1.05 2.1%
        4 1.15 1.05 3.8%
        8 1.32 1.05 5.2%
        12 1.58 1.05 7.1%
        16 1.85 1.05 8.5%

        位置越远，最优温度越高，标准温度的校准误差越大。

        3.3 Online Calibration (Kumar et al., 2024)

        论文：Online Calibration of Large Language Models

        核心贡献：

        1. LLM 的校准状态随输入分布变化
        2. 提出在线校准方法：持续更新温度参数
        3. 使用滑动窗口估计当前校准状态

          4. 四、硬件感知论文

          4.1 vLLM (Kwon et al., 2023)

        论文：Efficient Memory Management for LLM Serving with PagedAttention

        与硬件感知调度的关系：vLLM 的 continuous batching 机制为硬件感知调度提供了基础——调度器需要感知当前的批次大小和 GPU 状态。

        4.2 Orca (Yu et al., 2022)

        论文：Orca: A Distributed Serving System for Transformer-Based Generative Models

        核心贡献：

        1. Iteration-level scheduling
        2. 动态 batching 策略

          3. 与硬件感知的关系：Orca 的调度器需要考虑 GPU 的吞吐特性来决定何时合并请求。

          4.3 DSpark (核心论文)

          论文：DSpark: Efficient Speculative Decoding with Hardware-Aware Scheduling

          核心贡献：

          1. **置信度头**：轻量级分类器预估每个草稿位置的验证存活概率

            2. $$

            P_{\text{survive}}(i) = \sigma(w_c^T \cdot h_i + b_c)

            $$

            参数量仅 $d_{\text{hidden}} + 1$，计算成本可忽略。

            1. **吞吐量参考曲线**：系统启动时预先测算 GPU 在不同配置下的吞吐数据
            2. **动态最优求解**：在 GPU 内部实时求解最优验证长度

              3. $$

              K^* = \arg\max_K \frac{\sum_{i=1}^{K} P_{\text{survive}}(i) + 1}{T_{\text{verify}}(K, B) / \Theta(B)}

              $$

              4. 顺序温度缩放：为每个草稿位置使用独立的温度参数

              $$

              p^{(i)} = \text{softmax}(z^{(i)} / T_i)

              $$

              5. GPU 内部执行：整套调度逻辑完全在 GPU 上运行

              核心实验结果：

              七、关键技术细节补充

              7.1 置信度头的训练数据构造

              置信度头的训练需要构造正负样本：

              • 正样本：草稿 token 被目标模型接受的位置
              • 负样本：草稿 token 被目标模型拒绝的位置
                • 
def construct_training_data(draft_model, target_model, dataset):
    """构造置信度头训练数据"""
    data = []
    for batch in dataset:
        # 草稿生成
        draft_out = draft_model(batch["input_ids"])
        draft_hidden = draft_out.hidden_states[-1]
        
        # 目标模型验证
        target_out = target_model(batch["input_ids"])
        target_logits = target_out.logits
        
        # 判断接受/拒绝
        draft_probs = F.softmax(draft_out.logits, dim=-1)
        target_probs = F.softmax(target_logits, dim=-1)
        
        for i in range(batch["input_ids"].shape[1]):
            token = batch["input_ids"][0, i]
            ratio = target_probs[0, i, token] / (draft_probs[0, i, token] + 1e-8)
            accepted = 1 if torch.rand(1).item() < ratio.item() else 0
            
            data.append({
                "hidden": draft_hidden[0, i],
                "label": accepted
            })
    
    return data

                7.2 吞吐量曲线的拟合方法

            吞吐量曲线可以使用分段线性模型或高斯过程来拟合：

            $$

            \Theta(B, K) = \sum_{i} \alpha_i \cdot \phi_i(B, K)

            $$

            其中 $\phi_i$ 是基函数，$\alpha_i$ 是系数。
