KV Cache 核心论文深度解读

论文 1：Attention Is All You Need (Vaswani et al., 2017)

基本信息

  • 标题：Attention Is All You Need
  • 作者：Ashish Vaswani, Noam Shazeer, Niki Parmar, Jakob Uszkoreit, Llion Jones, Aidan N. Gomez, Lukasz Kaiser, Illia Polosukhin
  • 发表：NeurIPS 2017
  • DOI：10.48550/arXiv.1706.03762

论文核心内容

这篇论文提出了 Transformer 架构，完全基于注意力机制，抛弃了传统的循环和卷积结构。Transformer 的核心是自注意力机制（Self-Attention），也称为缩放点积注意力（Scaled Dot-Product Attention）：

Attention(Q, K, V) = softmax(QK^T / √d_k) V

其中 Q（Query）、K（Key）、V（Value）分别是输入通过三个不同的线性变换得到的矩阵。多头注意力（Multi-Head Attention）将 Q、K、V 分成 h 个头，各自独立计算注意力后拼接：

MultiHead(Q, K, V) = Concat(head_1, ..., head_h) W^O
where head_i = Attention(Q W_i^Q, K W_i^K, V W_i^V)

KV Cache 的隐含诞生

虽然论文本身并未明确提出"KV Cache"这一概念，但它在第 3.4 节"Self-Attention"中描述了自回归生成的关键约束：

"We also apply this to the self-attention layer in the decoder. The decoder can attend to all positions in the input sequence up to and including the current position. We need to prevent leftward information flow in the decoder to preserve the auto-regressive property. We implement this inside of scaled dot-product attention by masking out (setting to -∞) all values in the input of the softmax which correspond to illegal connections."

这个因果掩码（causal mask）机制意味着：在位置 t 的解码中，位置 i > t 的 token 不可见。因此，位置 1 到 t-1 的 K 和 V 在计算位置 t+1 时仍然有效——这就是 KV Cache 的理论基础。

在原始实现中，作者使用了"masked self-attention"，实际工程实现中自然会缓存已计算的 K 和 V。论文中的伪代码（Algorithm 1）虽然没有显式展示缓存逻辑，但任何高效的 Transformer 实现都会采用这一优化。

对 KV Cache 问题的预见

论文在讨论部分提到了一些与效率相关的观察： - 自注意力的时间复杂度为 O(n^2 · d)，其中 n 是序列长度 - 对于非常长的序列，这可能成为瓶颈 - 但在当时的主流应用场景（机器翻译，序列长度通常 < 100）中，这不是问题

这个"不是问题"的判断在 5 年后被彻底颠覆——当序列长度从 100 增长到 100,000+，自注意力的二次复杂度（以及 KV Cache 的线性内存增长）成为了大模型部署的核心挑战。

关键公式与数据

原始 Transformer 的配置：

参数 Base Model Big Model
层数 L 6 6
注意力头数 h 8 16
d_k = d_v 64 64
模型维度 d_model 512 1024
每 token KV (FP32) 8 KB 32 KB
序列长度 ~50-100 ~50-100
总 KV Cache ~0.4-0.8 MB ~1.6-3.2 MB

在原始论文的场景中，KV Cache 的内存占用完全可以忽略不计。这为后来的"内存危机"埋下了伏笔。

论文 2：Fast Transformer Decoding: One Write-Head is All You Need (Shazeer, 2019)

基本信息

  • 标题：Fast Transformer Decoding: One Write-Head is All You Need
  • 作者：Noam Shazeer
  • 发表：arXiv preprint arXiv:1911.02150
  • 意义：提出了 Multi-Query Attention (MQA)，KV Cache 优化的开山之作

论文核心内容

Shazeer 在这篇简短但影响深远的论文中提出了一个尖锐的问题：标准的多头注意力中，多个头真的都需要独立的 K 和 V 吗？

他观察到，在训练好的模型中，不同的注意力头往往学到了类似的模式——很多头实际上在做相似的事情。既然如此，为什么不共享 K 和 V，只保留不同的 Q？

Multi-Query Attention 的定义：

# 标准 MHA: h 组独立的 (Q, K, V)
Q_i = x W_i^Q, K_i = x W_i^K, V_i = x W_i^V  for i = 1..h

# MQA: h 组 Q，但只有 1 组 K, V
Q_i = x W_i^Q  for i = 1..h
K = x W^K       # 共享
V = x W^V       # 共享

head_i = Attention(Q_i, K, V)

KV Cache 缩减效果：

标准 MHA KV Cache: 2 × L × h × d_k × batch × seq_len
MQA KV Cache:      2 × L × 1 × d_k × batch × seq_len

缩减比 = h (通常为 8-32 倍)

实验结果

Shazeer 在机器翻译任务上进行了实验：

模型 翻译质量 (BLEU) 解码速度 KV Cache 大小
MHA (h=8) 28.4 1.0x 1.0x
MQA 28.1 1.3x 0.125x
MHA (h=1) 27.2 1.0x 0.125x

关键发现： 1. MQA 的质量损失极小（仅 0.3 BLEU），远好于将头数直接降到 1 2. 解码速度提升约 30%（在 2019 年的硬件上） 3. KV Cache 减少了 8 倍

深远影响

这篇论文在当时并未引起广泛关注，但其核心思想在 2022-2023 年大模型时代被重新发现并广泛应用： - PaLM (Google, 2022) 使用了 MQA - LLaMA-2 (Meta, 2023) 使用了 GQA（MQA 的推广） - Falcon (TII, 2023) 使用了 MQA - 几乎所有 2023 年之后的大模型都采用了某种形式的查询分组注意力

Shazeer 的洞见在于：他看到了 Transformer 推理效率的核心瓶颈在于 KV Cache 的内存访问，而非计算。减少 KV Cache 的大小是提升推理效率的最直接途径。

论文 3：Efficient Memory Management for Large Language Model Serving with PagedAttention (Kwon et al., 2023)

基本信息

  • 标题：Efficient Memory Management for Large Language Model Serving with PagedAttention
  • 作者：Woosuk Kwon, Zhuohan Li, Siyuan Zhuang, Ying Sheng, Lianmin Zheng, Cody Hao Yu, Joseph E. Gonzalez, Hao Zhang, Ion Stoica
  • 发表：SOSP 2023
  • 实现：vLLM - 目前最流行的大模型推理引擎之一

论文核心内容

这篇论文从系统层面重新审视了 KV Cache 的问题。与 MQA/GQA 从模型架构层面减少 KV Cache 大小不同，vLLM 关注的是：即使 KV Cache 的大小是固定的，如何更高效地管理这些内存？

核心观察：现有的 LLM 服务系统在 KV Cache 管理上存在严重的内存浪费：

  1. 预分配导致的内部碎片：为保证连续内存，系统按最大序列长度预分配 KV Cache。实际序列长度远小于最大值时，大量内存被浪费。
  2. 外部碎片：不同大小的 KV Cache 在显存中形成碎片，进一步降低利用率。
  3. 过度预留：为应对峰值负载，系统需要预留大量显存。

论文测量了现有系统（如 Orca）的内存利用率，发现： - 平均内存利用率仅为 20-40% - 高达 60-80% 的显存被浪费 - 这直接限制了能同时服务的请求数

PagedAttention 的设计：

借鉴操作系统虚拟内存的分页机制，PagedAttention 将 KV Cache 的管理分为两层：

逻辑层: 连续的 token 序列 [0, 1, 2, 3, 4, 5, ..., n-1]
物理层: 非连续的 block [block_7, block_3, block_12, block_1, ...]
映射:   block_table[i] = physical_block_id

每个 block 存储固定数量（如 16）个 token 的 KV Cache。block 不需要在物理内存中连续，通过 block table 进行地址映射。

关键特性：

  1. 按需分配：只在需要时分配新的 block，避免预分配浪费
  2. 零外部碎片：所有 block 大小相同，可以放入任何空闲位置
  3. 最小内部碎片：仅最后一个 block 有内部碎片，平均浪费 < 4%
  4. Copy-on-Write：多个序列可以共享相同的 KV block（如 beam search 中的不同候选），只在需要修改时才复制

代码示例（vLLM 核心逻辑简化）：

class PagedAttention:
    def __init__(self, block_size=16, num_gpu_blocks=1000):
        self.block_size = block_size
        self.block_table = {}  # seq_id -> [block_ids]
        self.free_blocks = set(range(num_gpu_blocks))

    def allocate_block(self, seq_id, position):
        """为新位置分配 KV block"""
        block_idx = position // self.block_size
        if block_idx >= len(self.block_table[seq_id]):
            # 需要新 block
            new_block = self.free_blocks.pop()
            self.block_table[seq_id].append(new_block)
        return self.block_table[seq_id][block_idx]

    def attention(self, q, seq_id, max_position):
        """使用分页 KV Cache 计算注意力"""
        blocks = self.block_table[seq_id]
        output = []
        for block_id in blocks:
            k_block = self.load_k(block_id)  # 从 GPU 内存加载
            v_block = self.load_v(block_id)
            attn_scores = q @ k_block.T / math.sqrt(self.d_k)
            output.append(softmax(attn_scores) @ v_block)
        return combine(output)

实验结果

vLLM 在多种模型和硬件配置上进行了评测：

系统 吞吐量 (相对值) 内存利用率 首 token 延迟
Orca 1.0x ~35% 基准
FasterTransformer 1.2x ~40% 基准
vLLM (PagedAttention) 2.4x ~96% 基准
vLLM + chunked prefill 3.2x ~96% +5%

在 GPT-3 175B 的推理场景中，vLLM 的吞吐量比 Orca 提升 2-4 倍，同时支持更大的 batch size。

深远影响

PagedAttention 的意义超越了 KV Cache 管理本身： 1. 它证明了 LLM 推理是一个系统问题，不仅仅是算法问题 2. 它启发了后续一系列推理优化工作（SGLang、TensorRT-LLM 等） 3. 它建立了"LLM serving as an OS problem"的研究范式

论文 4：DeepSeek-V2: A Strong, Economical, and Efficient Mixture-of-Experts Language Model (DeepSeek, 2024)

基本信息

  • 标题：DeepSeek-V2: A Strong, Economical, and Efficient Mixture-of-Experts Language Model
  • 作者：DeepSeek-AI
  • 发表：arXiv preprint arXiv:2405.04434
  • 核心创新：Multi-head Latent Attention (MLA) + MoE 架构

论文核心内容

DeepSeek-V2 的核心贡献之一是提出了 Multi-head Latent Attention (MLA)，这是一种从根本上重新设计 KV Cache 存储方式的注意力机制。

MLA 的核心思想：

传统注意力中，每一层需要缓存的 KV 大小为 2 × n_kv × d_k。MLA 通过低秩压缩将这一大小进一步降低：

# 压缩阶段（在每次前向传播时执行）
# 将隐藏状态投影到低维潜在空间
c_t = W_DKV · h_t
# W_DKV ∈ R^{d_c × d_model}, d_c 远小于 n_kv × d_k

# 缓存 c_t 而非完整的 K, V
# KV Cache per token = d_c (而非 2 × n_kv × d_k)

# 解压阶段（在计算注意力时执行）
K_t = W_UK · c_t  # W_UK ∈ R^{(n_heads × d_k) × d_c}
V_t = W_UV · c_t  # W_UV ∈ R^{(n_heads × d_v) × d_c}
Q_t = W_UQ · (W_DQ · h_t)  # Q 通过类似方式解压

压缩比分析：

标准 MHA: KV per token = 2 × h × d_k = 2 × 128 × 128 = 32,768
GQA (g=8): KV per token = 2 × g × d_k = 2 × 8 × 128 = 2,048
MLA: KV per token = d_c = 512 (DeepSeek-V2 配置)

MLA vs MHA 压缩比: 32,768 / 512 = 64x
MLA vs GQA 压缩比: 2,048 / 512 = 4x

MLA 的代价：

压缩和解压需要额外的计算。但论文指出，这些计算是计算密集型的矩阵乘法，可以高效地在 GPU 上执行。相比之下，KV Cache 的内存读取是内存带宽密集型的，是更大的瓶颈。因此，MLA 用少量额外计算换取了大量内存带宽节省，是一个净正收益。

实验结果

DeepSeek-V2 在多个维度上展示了 MLA 的效果：

模型 KV Cache (每 token) 训练 FLOPs MMLU HumanEval
LLaMA-2 70B (GQA) 2,048 1.0x 68.9 29.9
DeepSeek-V2 (MLA) 512 0.83x 70.5 45.1
压缩比 4x - - -

关键发现： 1. MLA 将 KV Cache 减少了约 93.3%（相比 MHA） 2. 相比 GQA，MLA 进一步减少了 4 倍 3. 模型质量不仅没有下降，反而有所提升（得益于更强的表达能力） 4. 推理时的吞吐量显著提升

MLA 与 MoE 的协同

DeepSeek-V2 的另一个关键设计是 MLA 与 MoE（Mixture of Experts）的协同： - MoE 减少了每次推理时的计算量（只激活部分专家） - MLA 减少了 KV Cache 的内存占用 - 两者结合，实现了"强且经济"的目标

DeepSeek-V2 的训练成本约为同等性能模型的 43.8%，推理成本约为 3.3%。

深远影响

MLA 的意义在于它展示了一种新的优化范式：不是在现有注意力机制上修修补补，而是从根本上重新设计 KV 的存储和计算方式。这种思路启发了后续一系列工作： - DeepSeek-V3 进一步优化了 MLA 的实现 - 多个研究组开始探索"可学习的 KV Cache 压缩" - MLA 的压缩-解压范式可能成为下一代注意力机制的标准组件
