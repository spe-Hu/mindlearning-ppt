硬件感知调度与在线校准 — 概念解剖（ljg-book 风格）

**核心问题**：每次应该猜几个 token？没有固定答案。

**精神内核**："越跑越准，真正自适应"

一、这本书在答什么问题？

1.1 问题的起源

推测解码将 LLM 推理分为"草稿"和"验证"两个阶段。草稿阶段生成 $K$ 个候选 token，验证阶段一次性检查。关键问题：$K$ 应该取多少？

固定 $K$ 的问题：

  • $K$ 太大 → 浪费验证计算（大部分被拒绝）
  • $K$ 太小 → 限制加速效果（每轮前进太少）

    • 1.2 问题的深化

    最优 $K$ 不是常数，它取决于：

    1. **输入内容**：常见短语 vs 罕见词汇
    2. **草稿模型状态**：高置信度 vs 低置信度
    3. **GPU 负载**：空闲 vs 满载

4. 批次大小：单请求 vs 多请求

1.3 问题的极致

DSpark 的回答是：让系统自己决定。通过置信度头预估每个位置的存活概率，结合 GPU 吞吐量曲线，实时求解最优 $K$。同时用顺序温度缩放持续校准，让系统"越跑越准"。

二、立在什么不证之命题上？

2.1 第一公理：可预测性假设

假设：草稿位置的验证存活概率是可以被预测的。

置信度头通过学习草稿模型的隐藏状态与验证结果之间的关系，可以较准确地预估每个位置的存活概率。

$$

P_{\text{survive}}(i) \approx f(h_i^{\text{draft}})

$$

如果存活概率完全不可预测，置信度头就失去了意义。

2.2 第二公理：硬件稳定性假设

假设：GPU 的吞吐量在短时间内是相对稳定的。

预先测算的吞吐量参考曲线在短时间内（几分钟到几小时）是有效的。如果 GPU 的吞吐量剧烈波动，参考曲线就失去了指导意义。

2.3 第三公理：校准连续性假设

假设：分布偏差是连续变化的，不是突变的。

顺序温度缩放假设相邻位置的最优温度是相近的。如果温度在不同位置之间剧烈跳变，顺序缩放就失去了效果。

三、用什么框架看世界？

3.1 收益-成本优化框架

$$

K^* = \arg\max_K \frac{\text{Benefit}(K)}{\text{Cost}(K)}

$$

其中：

  • $\text{Benefit}(K) = \sum_{i=1}^{K} P_{\text{survive}}(i) + 1$（期望接受长度）
  • $\text{Cost}(K) = T_{\text{verify}}(K, B) / \Theta(B)$（验证成本）

    • 3.2 校准-精度权衡框架

    $$

    \min_T \text{ECE}(T) \quad \text{s.t.} \quad \text{Accuracy}(T) \geq \text{threshold}

    $$

    3.3 在线学习框架

    $$

    T_i^{(t+1)} = T_i^{(t)} - \eta \nabla_{T_i} \mathcal{L}_{\text{NLL}}^{(t)}

    $$

    每个位置的温度参数通过在线梯度下降持续更新。

    四、得出什么结论？

    4.1 核心结论

    1. **最优草稿长度是动态的**：不存在一个固定的最优 $K$，它随输入、模型状态和硬件负载变化。
    2. **硬件感知是必须的**：不同 GPU 的最优 $K$ 不同，同一 GPU 在不同负载下的最优 $K$ 也不同。
    3. **在线校准是持续的**：分布偏差随时间变化，需要持续校准。

4. GPU 内部执行是关键：CPU-GPU 传输是隐蔽的性能瓶颈。

4.2 推论

  1. **置信度头是"预言家"**：它能提前知道哪些草稿 token 会被接受，从而避免浪费验证计算。
  2. **吞吐量曲线是"地图"**：它告诉调度器在不同配置下的验证成本，从而做出最优决策。
  3. **顺序温度是"微调器"**：它为每个位置定制校准参数，将 ECE 从 3%-8% 压到 ~1%。

    4. 五、萃出一个 Takeaway

    取景框

"越跑越准，真正自适应"

这十个字浓缩了硬件感知调度的全部精髓：

  • **越跑越准**：在线校准持续优化温度参数
  • **真正自适应**：不仅适应输入变化，还适应硬件状态变化

    • 模型

    $$

    \text{DSpark} = \underbrace{f_{\text{conf}}}_{\text{置信度头}} + \underbrace{\Theta(B)}_{\text{吞吐曲线}} + \underbrace{T_i}_{\text{顺序温度}} + \underbrace{\text{GPU}}_{\text{内部执行}}

    $$

    洞见

    "最优不是预设的，而是涌现的"

    传统的调度方案试图预设一个"最优"配置。DSpark 的哲学是让最优配置在运行中自然涌现——通过持续感知、持续学习、持续适应。

"""
硬件感知调度与在线校准完整实现
"""
import torch
import torch.nn as nn
import torch.nn.functional as F

class ConfidenceHead(nn.Module):
    """置信度头：预估每个草稿位置的验证存活概率"""
    
    def __init__(self, hidden_dim=4096):
        super().__init__()
        self.classifier = nn.Linear(hidden_dim, 1)
    
    def forward(self, hidden_states):
        """
        Args:
            hidden_states: [K, hidden_dim] 草稿模型的隐藏状态
        Returns:
            survival_probs: [K] 每个位置的存活概率
        """
        logits = self.classifier(hidden_states).squeeze(-1)
        return torch.sigmoid(logits)


class ThroughputCurve:
    """GPU 吞吐量参考曲线"""
    
    def __init__(self):
        self.table = {}  # (batch_size, K) -> throughput
    
    def calibrate(self, model, device):
        """预先测算吞吐量"""
        for B in [1, 2, 4, 8, 16, 32]:
            for K in [4, 8, 12, 16, 24, 32]:
                # 测量吞吐量
                input_ids = torch.randint(0, 32000, (B, K)).to(device)
                # 预热 + 测量
                for _ in range(10): model(input_ids)
                torch.cuda.synchronize()
                start = torch.cuda.Event(enable_timing=True)
                end = torch.cuda.Event(enable_timing=True)
                start.record()
                for _ in range(100): model(input_ids)
                end.record()
                torch.cuda.synchronize()
                elapsed = start.elapsed_time(end) / 100  # ms
                throughput = B * K / (elapsed / 1000)  # tokens/s
                self.table[(B, K)] = throughput
    
    def query(self, batch_size, K):
        """查询吞吐量"""
        return self.table.get((batch_size, K), 1000)


class SequentialCalibrator:
    """顺序温度缩放校准器"""
    
    def __init__(self, max_K=32):
        self.temperatures = nn.Parameter(torch.ones(max_K))
    
    def calibrate(self, logits, position):
        """校准指定位置的 logits"""
        T = self.temperatures[position]
        return F.softmax(logits / T, dim=-1)
    
    def update(self, position, logits, target_probs, lr=0.01):
        """在线更新温度"""
        T = self.temperatures[position]
        log_probs = F.log_softmax(logits / T, dim=-1)
        loss = -(target_probs * log_probs).sum(-1).mean()
        loss.backward()
        with torch.no_grad():
            self.temperatures[position] -= lr * self.temperatures[position].grad
            self.temperatures[position].grad = None
            self.temperatures[position].clamp_(0.5, 3.0)


class HardwareAwareScheduler:
    """硬件感知调度器"""
    
    def __init__(self, confidence_head, throughput_curve, calibrator):
        self.conf_head = confidence_head
        self.throughput = throughput_curve
        self.calibrator = calibrator
    
    def schedule(self, draft_hidden, batch_size):
        """
        动态求解最优验证长度
        
        全在 GPU 上执行，无 CPU 开销
        """
        K_max = draft_hidden.shape[0]
        
        # 1. 预估存活概率
        survival_probs = self.conf_head(draft_hidden)
        
        # 2. 计算每个 K 的净收益
        gains = torch.zeros(K_max, device=draft_hidden.device)
        cum_accept = 0.0
        for K in range(K_max):
            cum_accept += survival_probs[K].item()
            cost = self.throughput.query(batch_size, K + 1)
            gains[K] = (cum_accept + 1) / cost
        
        # 3. 选择最优 K
        K_opt = gains.argmax().item() + 1
        
        return K_opt

    八、深度数学分析

    8.1 调度优化的对偶问题

    原始问题：

    $$

    \max_K \frac{\sum_{i=1}^{K} P_{\text{survive}}(i) + 1}{T_{\text{verify}}(K, B) / \Theta(B)}

    $$

    对偶问题：

    $$

    \min_\lambda \lambda \cdot T_{\max} + \sum_{K} \max\left(0, \text{Benefit}(K) - \lambda \cdot \text{Cost}(K)\right)

    $$

    其中 $\lambda$ 是对偶变量，表示时间的"价格"。当 GPU 繁忙时，$\lambda$ 增大，调度器倾向于选择更小的 K。

    8.2 顺序温度缩放的最优性证明

    定理：对于草稿序列中位置相关的分布偏差，顺序温度缩放优于全局温度缩放。

    证明（概要）：

    设位置 $i$ 的最优温度为 $T_i^$，全局最优温度为 $T^$。全局温度缩放的校准误差为：

    $$

    \text{ECE}_{\text{global}} = \sum_i w_i |T^ - T_i^|

    $$

    顺序温度缩放的校准误差为：

    $$

    \text{ECE}_{\text{seq}} = 0 \quad (\text{因为每个位置都用最优温度})

    $$

    因此 $\text{ECE}_{\text{seq}} \leq \text{ECE}_{\text{global}}$，等号成立当且仅当所有 $T_i^*$ 相等。

    8.3 在线学习的遗憾界

    对于在线梯度下降，遗憾界为：

    $$

    R_T = \sum_{t=1}^{T} \ell_t(T^{(t)}) - \min_{T^} \sum_{t=1}^{T} \ell_t(T^) \leq \frac{D^2}{2\eta} + \frac{\eta G^2 T}{2}

    $$

    取 $\eta = D / (G\sqrt{T})$，得到 $R_T \leq DG\sqrt{T}$，即平均遗憾为 $O(1/\sqrt{T})$。
